# v0.4.0 Design: Complete Microseg

**Date:** 2026-02-18
**Goal:** Make segspec standalone viral for platform engineers and DevOps by shipping production-complete output, real-world config coverage, and a trust layer.
**Approach:** "Complete Microseg" — ingress first, then Helm, then interactive TUI.

## Target Audience

Platform engineers running K8s clusters + Developers/DevOps who get asked to "add network policies." They share segspec when it saves them weeks of manual policy writing and removes a hated chore.

## Feature 1: Per-Service Ingress Policies

### Problem

segspec v0.3.0 generates egress-only policies — "what does my app talk TO?" Without ingress, it's half a solution. Platform engineers can't enforce real microseg.

### Design

The dependency graph is already bidirectional in the data. If `frontend -> cartservice:8080`, then `cartservice` has an inbound from `frontend` on port 8080. The renderer inverts the graph.

- `DependencySet` gets an `IngressFor(service)` method — returns all deps where the target matches the given service name
- Renderer generates N policies (one per discovered service) instead of one monolithic policy
- Each policy contains egress rules (existing), ingress rules (new, derived by inverting), and default-deny for both directions
- Services identified by name from K8s manifests, Docker Compose service names, or Spring application names

### Output Example

```yaml
# cartservice
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: cartservice-netpol
spec:
  podSelector:
    matchLabels:
      app: cartservice
  policyTypes: [Ingress, Egress]
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: frontend
      ports:
        - port: 8080
  egress:
    - to:  # redis-cart
      ports:
        - port: 6379
```

## Feature 2: Helm Template Resolution

### Problem

Most production K8s apps use Helm. Templates with `{{ .Values.x }}` placeholders can't be parsed directly.

### Design

Shell out to `helm template` to render the chart into plain YAML, then feed rendered output to the existing K8s parser. segspec doesn't understand Helm templating — it just needs Helm to resolve it.

- Walker detects Helm charts by finding `Chart.yaml` in directories
- Runs `helm template <chart-dir>` with 30s timeout
- Pipes rendered YAML into existing K8s parser — no new parser needed
- Optional `--helm-values values-prod.yaml` flag for specific values files (defaults to chart's built-in `values.yaml`)
- Graceful fallback: if `helm` binary isn't installed, print stderr warning and skip

### What segspec does NOT do

- No Helm install/repo management
- No Tiller interaction
- No chart downloading (GitHub URL support handles cloning)

### Edge Cases

- Multiple charts in one repo: each gets `helm template` separately
- Charts with required values that have no defaults: warning, skip that chart
- Subchart dependencies: `helm template` resolves these automatically

## Feature 3: Interactive TUI

### Problem

With ingress + Helm expanding coverage, segspec might find 30-50+ deps. Platform engineers won't blindly apply that.

### Design

Terminal UI using charmbracelet/bubbletea (standard Go TUI library, used by `gh`, `gum`, `soft-serve`). Activated via `--interactive` or `-i` flag.

### Flow

1. Discovery phase runs as normal (parsers + optional AI)
2. TUI launches with table of all discovered deps
3. All deps start checked — user unchecks what they don't want
4. Keybindings: arrow/jk navigate, Space toggle, e edit, a/n all/none, f filter, Enter confirm, q quit
5. After confirm: generates YAML from only checked deps

### Key Decisions

- Low-confidence (AI-discovered) deps are visually distinct
- Non-interactive mode remains the default — TUI is opt-in
- If stdout is not a TTY (piped/CI), `--interactive` warns and falls back to normal output

## Ship Order

1. Per-service ingress policies (unlocks production deployment)
2. Helm template resolution (expands real-world coverage)
3. Interactive TUI (adds trust layer)

## Non-Goals for v0.4.0

- GitHub Action (roadmap for v0.5.0)
- Multi-format output (AWS SGs, Cilium, Calico)
- Helm chart downloading or repo management
- Browser-based UI
